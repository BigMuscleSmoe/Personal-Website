URL Shortener

Flask REST API · Caching & Analytics
Overview

A URL shortener REST API built with Flask, PostgreSQL, and Redis — generating short links, handling tracked redirects, and serving click analytics. The design leans on cache-aside lookups, composite database indexing, async click recording, and Redis-backed rate limiting to stay read-fast under load.

Endpoints
POST /api/links
Create a short link from a JSON { "url": ... } body → 201 with code + short URL. Rate-limited to 10/min per IP (429 on excess).
GET /{code}
Redirect to the original URL via 302 Found so every visit is tracked. 404 if the code is unknown.
GET /api/links/{code}/stats
Analytics: total clicks, daily breakdown, and top referrers with their click counts.
docker compose up --build
Brings up Flask, PostgreSQL, and Redis together at localhost:8080.

How it works

The real service is Flask + PostgreSQL + Redis running in Docker, with codes cached for fast redirects and clicks written asynchronously. This browser demo reimplements the same request/response contract in JavaScript — base62 code generation, cache-aside lookups, 302 redirects, async click recording, rate limiting, and the stats aggregation — entirely client-side. No network involved.

Features
Short Code Strategy
Seven random base62 characters (~3.5 trillion combinations) with collision-retry. Codes stay unpredictable so they don't leak usage patterns.
Tracked Redirects
302 Found (not 301) ensures every visit reaches the server so each click is counted, instead of being cached permanently by the browser.
Redis Caching
Cache-aside redirect lookups with a 24-hour TTL and a 5-minute negative cache for unknown codes — keeping hot paths off the database.
Async Click Writes
Click rows are inserted on background threads, letting redirect responses return immediately without waiting on the database.
Analytics
Total clicks, daily breakdown, and top referrers, served by a composite index on clicks(link_id, clicked_at) for read-heavy queries.
Rate Limiting
Flask-Limiter backed by Redis enforces 10 creates/min per IP across multiple app instances, returning 429 on excess.
Highlights

What I Built

  • Designed a Flask REST API for link creation, tracked redirects, and click analytics, served by Gunicorn behind Docker Compose
  • Implemented base62 code generation with collision-retry and a unique constraint on the code column
  • Built cache-aside redirect lookups in Redis with a 24-hour TTL and a short negative cache for unknown codes
  • Moved click recording onto background threads so redirects return without blocking on database writes
  • Tuned analytics with a composite index on clicks(link_id, clicked_at) for total, daily, and top-referrer queries
  • Added Redis-backed rate limiting (10/min per IP) and pytest tests running against an in-memory SQLite database
